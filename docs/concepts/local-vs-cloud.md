# 本地控制 vs 雲端控制

這是整個米家生態最關鍵、也最容易踩雷的一條軸。你選的每一種控制方式（[CLI](../control/cli.md)、[MCP](../control/mcp.md)、[Siri 橋接](../ecosystem/apple-home.md)…）背後，都是走「本地」或「雲端」其中一條路，這決定了**延遲、可靠度、隱私，以及哪些裝置根本控不到**。

!!! note "一句話"
    **本地 = miIO/LAN 協議 + 裝置 token**；**雲端 = MIoT Cloud + 帳號 QR 登入**。認證方式基本上就對應了這條軸。

## 對照表

| 面向 | 本地控制（miIO / LAN） | 雲端控制（MIoT Cloud） |
|---|---|---|
| 認證方式 | 裝置 **token**（見 [認證與 Token](auth-token.md)） | 小米帳號 **QR 登入** |
| 連線路徑 | 區網內直達裝置 | 經小米雲端伺服器中轉 |
| 延遲 | 低（毫秒級） | 較高，依網路與伺服器 |
| 網路需求 | 只要區網、可離線 | 需要網際網路 |
| 裝置覆蓋 | 只限支援 miIO 本地的裝置 | 帳號看得到的幾乎都可控 |
| BLE / 紅外線裝置 | ❌ 天生不支援 | ✅ 只能走雲端 |
| 隱私 | 指令不出區網 | 每個指令都經過小米 |
| 穩定性風險 | 新韌體可能鎖本地協議 | 依賴小米雲端與登入不被改版 |

## 重點細節

- **BLE / 紅外線裝置是雲端限定**：藍牙溫濕度計、紅外遙控類設備沒有 IP、沒有 miIO 本地介面，只能透過雲端（或經一個會把它們代理出來的網關）。
- **新款 MIoT 裝置越來越「雲端化」**：早年米家設備多可用 miIO 在區網直控，但越來越多新裝置把本地協議加密或停用，實際退化成只能走雲端。買新裝置前不要假設一定能本地控。
- **「本地」不等於「不用小米帳號」**：多數本地方案（例如 [`python-miio`](../control/cli.md)）仍需要先用帳號**取一次 token**，之後才能離線直控。
- **完整本地控制通常需要網關**：把一整組設備穩定地本地化，實務上要靠 Xiaomi Central Hub Gateway（中央網關），而它的完整本地能力**目前主要在中國大陸區**——這牽涉到[帳號分區](account-region.md)。

## 這條軸怎麼影響你的選擇

| 你在意 | 傾向 |
|---|---|
| 低延遲、離線可用、隱私 | 本地為主（需處理 token、可能受新韌體限制） |
| 覆蓋率最高、設定最省事 | 雲端為主（QR 登入即可，但依賴網路/小米） |
| 又要廣覆蓋又要盡量本地 | 上 [Home Assistant](../solutions/index.md) + 中央網關，混合兩者 |

> 下游每個工具頁都會標註它走本地還是雲端；看到「token」想到本地，看到「QR 登入」想到雲端。
