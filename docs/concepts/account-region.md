# 帳號分區與地區限制

小米帳號是**分區（region-locked）**的。這件事在台灣使用者身上特別容易踩雷，因為它會同時影響 token 抓取、官方整合能不能連、以及 Google 綁定。

## 為什麼會有分區

小米的雲端服務依伺服器地區切分，常見的有 **中國大陸 / 新加坡 / 歐洲 / 美國 / 印度** 等。你的帳號、裝置與 App 都綁在某一個伺服器區：

- **台版設備**通常落在**非中國大陸區**（多為新加坡或自選的地區）。
- 同一顆帳號在不同區看到的裝置清單、可用功能、甚至韌體可能不同。
- 換區有時要重置裝置、重新配網。

## 分區會影響什麼

| 影響面向 | 說明 |
|---|---|
| token 抓取 | `miiocli cloud` / 抽取工具都要**選對 region** 才列得到裝置與 token（見 [認證與 Token](auth-token.md)） |
| 官方 HA 整合 | [`ha_xiaomi_home`](../solutions/home-assistant.md) 的 OAuth 登入要對應到裝置所在區 |
| Google 綁定 | 小米帳號區與 Google 帳號區不一致時，[綁定](../ecosystem/google-home.md)常失敗或裝置不顯示 |
| 完整本地控制 | 靠中央網關的完整[本地](local-vs-cloud.md)方案**目前主要在中國大陸區**，其他區支援有限 |

!!! warning "動手前先確認你的區"
    設定任何整合前，先確定你的米家 App 帳號在哪一個伺服器區（App 設定裡可看／可切）。抓 token、裝官方整合、綁 Google 時都要用**同一個區**，否則會出現「登入成功但看不到裝置」這類鬼打牆。

## 對台灣使用者的建議

- 記錄下你帳號目前的區，之後所有工具都用它。
- 若你的裝置在新加坡區，別預期能立刻拿到中央網關的完整本地控制；把「本地化」當成加分項而非前提。
- 需要跨區合併裝置時，[Home Assistant](../solutions/index.md) 這類中樞能幫你把不同來源正規化到同一個介面。
